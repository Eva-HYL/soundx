# 财务规则

## 1. 基本原则

本系统采用人工支付模式：
- 不接微信支付或支付宝 API
- 不承担资金托管
- 不做自动分账
- 所有支付确认和转账动作都必须留痕

## 2. 订单付款确认

- 用户根据俱乐部提供的收款方式完成线下转账
- 管理员核对到账后，在系统内执行“确认付款”
- 确认付款后订单才允许进入派发流程
- 系统需记录确认人、确认时间、确认金额、备注和凭证

## 3. 积分结算

- 陪玩收入通过积分表达，不直接体现为系统余额
- 管理员审核通过报备后，为陪玩增加积分
- 每次积分变动必须写入积分流水
- 积分流水是结算追溯的唯一依据

建议流水类型：
- `order_income`
- `reward_income`
- `withdraw_freeze`
- `withdraw_deduct`
- `withdraw_unfreeze`
- `manual_adjust`
- `penalty`

## 4. 提现规则

- 陪玩可对可提现积分发起提现申请
- 提现申请创建后，应冻结对应积分
- 管理员审核通过后执行线下转账
- 转账完成后，在系统中确认提现完成并扣减冻结积分
- 驳回提现时，需解冻对应积分

建议提现状态：
- `pending_review`
- `approved_pending_transfer`
- `transferred_pending_confirm`
- `completed`
- `rejected`
- `cancelled`

## 5. 打赏规则

- 打赏本质上是订单相关的额外收入
- 打赏若进入 MVP，可复用人工支付确认机制
- 打赏收入如需计入陪玩收益，也必须走积分流水

## 6. 审计要求

以下动作必须产生独立记录：
- 订单付款确认
- 打赏付款确认
- 提现审核
- 提现转账确认
- 积分手工调整

系统应至少保存：
- 操作人
- 业务对象
- 动作类型
- 动作前后数据
- 操作时间
